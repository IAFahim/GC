## 🚨 PRIORITY 0: CRITICAL INTEGRATION & SETUP FAILURES (Nautilus Sabotage Fixes)
- [ ] **Fix `setup.sh` sed Replacement Failure:** The command `sed "s|^gc |$GC_PATH |g"` strictly matches `gc ` at the start of a line. In `gc-nautilus.sh`, the execution line is `OUTPUT=$(gc --paths...`. The replacement silently fails, leaving the script calling an unresolvable `gc` binary. **Fix:** Use `sed "s|OUTPUT=\$(gc |OUTPUT=\$($GC_PATH |g"` or inject `export PATH="$HOME/.local/bin:$PATH"` at the top of the Nautilus script.
-[ ] **Fix `url_decode` Format String Vulnerability:** In `gc-nautilus.sh`, `printf '%b' "${url_encoded//%/\\x}"` is used. If a selected filename contains native bash escape sequences (like `\n`, `\t`, or `\033`), `printf '%b'` will evaluate them, permanently mangling the file path and causing silent failures. **Fix:** Use Python or Perl for safe URL decoding, or implement a strictly sanitized bash hex-decoder.
- [ ] **Fix Git Root Resolution in Nautilus:** The `while [ "$CURRENT_DIR" != "/" ]` loop correctly finds `.git`, but if the user selects multiple files spanning *different* repositories, the script forces them all into the first file's working directory. **Fix:** Group `TARGET_PATHS` by their respective Git roots and execute `gc` separately per repository context, or ensure `gc` processes absolute paths independently of `pwd`.
- [ ] **Fix Space/Quote Handling in Nautilus Array:** `OUTPUT=$(gc --paths -- "${TARGET_PATHS[@]}" 2>&1)` correctly handles spaces, but if `notify-send` receives a `$STATS` or `$ERROR_MSG` string containing double quotes or backticks from the gc output, the GUI notification will silently drop or crash. **Fix:** Quote variables strictly: `notify-send "gc" "${STATS}"`.

## 💥 PRIORITY 1: SEVERE PERFORMANCE & MEMORY CASCADES (The "100MB+ LOH" Issues)
- [ ] **Fix `GetSafeFence` Full-File Read Exhaustion:** In `MarkdownGeneratorExtensions.cs`, if `content.Content == null` (streaming mode), `GetSafeFence` opens a `StreamReader` and reads the *entire file* to the end just to count backticks. If a user includes a 5GB log file, `gc` reads 5GB synchronously before writing a single byte. **Fix:** Read only the first 8192 bytes to search for backticks, or default to a safe `````````` fence for streamed files larger than 1MB.
- [ ] **Fix O(N) Process Spawn Death Loop in Filesystem Discovery:** In `GitDiscoveryExtensions.cs`, `ShouldIgnoreFile` calls `Process.Start("git check-ignore...")` for *every single file* in the directory tree. Scanning a `node_modules` folder with 50,000 files spawns 50,000 separate `git` OS processes, creating an instant system lockup/fork-bomb. **Fix:** Buffer discovered paths and call `git check-ignore --stdin` in batches of 1,000, or implement a native `.gitignore` parser (e.g., the `Ignore` NuGet package).
- [ ] **Fix False Memory Limit Block on Streaming:** In `FileReaderExtensions.cs`, `ReadContentsLazy` calculates `estimatedMemorySize = totalSize * 2;` and immediately triggers `Environment.Exit(1)` if it exceeds `MaxMemoryBytes`. Streaming mode uses almost zero memory regardless of total file size! This entirely defeats the purpose of streaming. **Fix:** Remove the memory limit exit condition for `ReadContentsLazy`.
- [ ] **Eliminate String Allocation for Clipboard Output:** `Program.cs` calls `markdown.HandleOutput(cliArgs, fileContents)`. The `markdown` string holds the entire generated text (triggering Large Object Heap fragmentation). **Fix:** Refactor `ClipboardExtensions.cs` to accept an `Action<Stream>` or `IEnumerable<string>`. Pipe the data directly into the standard input stream of the clipboard process (`pwsh`, `pbcopy`, `wl-copy`) asynchronously.
- [ ] **Fix Double Disk I/O in Select Pipeline:** In `FileFilterExtensions.cs`, the `.Select` block creates `new FileEntry` and calls `File.Exists(path)` followed by `new FileInfo(path).Length`. This hits the disk twice. **Fix:** Instantiate `var fileInfo = new FileInfo(path);` and check `if (fileInfo.Exists)` to halve disk I/O operations.

## 🔒 PRIORITY 2: SECURITY, THREAD SAFETY & CORRUPTION VULNERABILITIES
- [ ] **Fix Infinite Symlink Recursion:** In `GitDiscoveryExtensions.cs`, `visitedPaths.Add(realPath)` uses `Path.GetFullPath()`, which resolves `../` but **does not** resolve symlinks. A circular symlink will cause an infinite loop until `OutOfMemoryException` crashes the app. **Fix:** Use `DirectoryInfo.ResolveLinkTarget(true)` to track actual INode/Target paths in `visitedPaths`.
- [ ] **Fix Missing Trailing Newline Corruption:** In `WriteMarkdownToStream`, if a streamed file does not end with a newline character, the closing ``` fence is appended directly to the last line of code (e.g., `return true;````), breaking the Markdown format. **Fix:** Unconditionally write a `\n` before the closing fence.
- [ ] **Fix File Locking Thread Crash:** In `TryReadFile`, `var text = reader.ReadToEnd();` will throw an `IOException` if the file is locked exclusively by another process (e.g., an active database or log file). **Fix:** Catch `IOException`, increment `errorCount`, log the specific locked file, and return `null` instead of crashing the entire PLINQ thread.
-[ ] **Fix Clipboard Encoding Corruption on Windows:** `ClipboardExtensions.CopyToClipboardWindowsClipExe` sets `StandardInputEncoding = Encoding.UTF8`, but `cmd.exe /c clip.exe` relies on the system locale code page. **Fix:** Ensure PowerShell 7+ is the primary mechanism and pass UTF-8 cleanly via `-Command "[Console]::InputEncoding =[System.Text.Encoding]::UTF8; $input | Set-Clipboard"`.
- [ ] **Fix Unhandled `OutOfMemoryException` Masking:** `TryReadFile` has `catch (OutOfMemoryException) { throw; }` but the outer `Program.cs` uses `catch (Exception ex)` and exits with code 1. This prevents the .NET runtime from properly unwinding or generating an OOM crash dump. **Fix:** Never catch `OutOfMemoryException`; catch `Exception` selectively.

## 🛠 PRIORITY 3: CORE LOGIC & ACCURACY BUGS
- [ ] **Fix Extension Truncation Bug:** In `FileFilterExtensions.cs`, `GetFullExtension` uses `.Split('.')` and skips to the first dot. If a file is named `app.controller.ts`, it extracts `controller.ts`. If the user provided `--extension ts`, it fails to match. **Fix:** Use `Path.GetExtension(path).TrimStart('.')` for strict matching, and maintain the split logic only for multi-part matches like `.tar.gz`.
- [ ] **Fix Default BOM Injection in File Output:** In `ClipboardExtensions.cs` (line 39), `File.WriteAllText(args.OutputFile, markdown);` defaults to UTF-8 **with BOM**. This breaks LLM context ingestion pipelines that expect strict POSIX `UTF-8-NOBOM`. **Fix:** Use `File.WriteAllText(args.OutputFile, markdown, new UTF8Encoding(false));`.
- [ ] **Fix CancellationToken Ignorance:** In `GenerateMarkdownStreaming`, the loop processes files without ever checking `cts.Token.ThrowIfCancellationRequested()`. If the user hits `Ctrl+C` while streaming 10GB of data, the process continues writing until completion. **Fix:** Add `args.CancellationToken.ThrowIfCancellationRequested();` inside the `foreach` loop.
- [ ] **Fix `TotalBytes` Calculation Bug:** In `TryReadFile` (streaming mode), `fileLength` is extracted from `FileInfo`. However, in `GenerateMarkdownStreaming`, `StreamReader` reads characters. Depending on the encoding (e.g., UTF-8 multi-byte characters), the bytes written to the output stream will not match `content.Size`, rendering the CLI's total size summary inaccurate. **Fix:** Track bytes explicitly during the `StreamReader` loop.

## 🧪 PRIORITY 4: CI/CD & TESTING INFRASTRUCTURE
- [ ] **Fix Global State Leak in Tests:** `NonGitDiscoveryTests.cs` initializes `_testDir = Path.Combine(Path.GetTempPath(), ...)`. However, concurrent tests writing to temporary directories can overlap if OS path generation collides, or if tests mutate global environment variables. **Fix:** Isolate all IO tests using uniquely seeded GUIDs per `[Fact]` execution, not just per class instance.
- [ ] **Fix GitHub API Rate-Limit Flakiness:** In `ReleaseBinaryTests.cs`, the mock handler successfully intercepts `api.github.com`. However, the test `CheckLatestReleaseExists` uses `EnsureSuccessStatusCode()` on the response. The mock handler is currently hardcoded to return a fake JSON string. Ensure the mock perfectly simulates the exact schema expected by the extraction logic, or the deserialization step will throw.
- [ ] **Fix `TestRunner.cs` Subprocess Hanging:** `TestRunner.cs` uses `process.StandardOutput.ReadLine()` synchronously in a `while` loop, followed by `StandardError.ReadLine()`. If the test process writes enough to `StandardError` to fill the OS buffer while the runner is blocked reading `StandardOutput`, a classic deadlock occurs. **Fix:** Use `BeginOutputReadLine()` and `BeginErrorReadLine()` with asynchronous event handlers.
- [ ] **Remove `mcr.microsoft.com/dotnet/sdk:10.0` from Native AOT Build:** The `Dockerfile` uses the standard SDK to build Native AOT but relies on `apt-get install clang gcc`. Alpine/Chiseled images require specific static compilation flags (`-p:PublishAot=true -p:StaticExecutable=true`). Ensure the build stage explicitly handles static linking for the target runtime OS to prevent `glibc` mismatch panics on target systems.
